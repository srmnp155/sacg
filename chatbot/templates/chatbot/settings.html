Local Prerequisites

  1. Install VS Code (or any IDE) and required runtime.
  2. Use chat/test/publish pages to generate and download code ZIP.
  3. Extract project and open folder in your local IDE.
  4. Create a virtual environment and install project dependencies.

How Features Work

  • Chat: generate/update code in a session.
  • Test: inspect and run saved session files.
  • GitHub: push session code to your GitHub repository.
  • Publish: use ZIP/GitHub deployment options from publish page.

Where To Set GitHub Token

  1. Open Profile.
  2. Set GitHub Username and GitHub Token.
  3. Save profile and use GitHub button from Chat/Test/Publish pages.

Azure Credentials (For Future Azure Mode)

  1. Open Azure Portal -> Microsoft Entra ID -> App registrations.
  2. Create or open your app registration.
  3. Copy Application (client) ID and Directory (tenant) ID.
  4. Go to Certificates & secrets, create client secret, copy Value (not Secret ID).
  5. Grant RBAC access in Subscription/Resource Group (Contributor or scoped role).
{% else %}

Azure App Service Prerequisites

  1. Create Resource Group and App Service Plan.
  2. Create a Web App (Linux, Python runtime as needed).
  3. Enable Managed Identity on your Django App Service.
  4. Grant RBAC role (Contributor) on target Resource Group.
  5. Set required App Settings (`DJANGO_*`, `AZURE_OPENAI_*`).

Azure Deploy Flow

  1. Generate and save code in a chat session.
  2. Open session Publish page.
  3. Choose Deployment using zip or Deployment using GitHub.
  4. Provide Subscription ID, RG, Plan, Web App Name, Region.
  5. Submit publish and track job status until URL is returned.

Azure App Service Mode

Use this mode when your app service identity and permissions are configured.
